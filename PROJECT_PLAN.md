# 9999: Rocket Escape 기획서

## 1. 프로젝트 개요

`9999: Rocket Escape`는 정사각형 우주 보드 안에서 로켓을 조종해 별을 먹고 장애물을 피하는 흑백 Canvas 미니게임이다. Snake 게임의 제한된 공간 긴장감을 로켓 회피 액션으로 재해석했고, 점수가 오를수록 로켓과 장애물이 함께 빨라지는 구조로 짧은 플레이 안에서도 속도감과 재도전 욕구를 만든다.

- 게임명: 9999: Rocket Escape
- 장르: 흑백 레트로 아케이드 생존 게임
- 플랫폼: 웹 브라우저
- 핵심 조작: 방향키 또는 WASD
- 핵심 목표: 별을 먹으며 가능한 오래 생존하고 최고 점수에 도전
- 제작 방식: Vite, TypeScript, HTML Canvas, CSS

## 2. 대회 적합성

월간 해커톤 “웹 미니게임 챌린지”는 사용자가 짧은 시간 안에 규칙을 이해하고 바로 반복 플레이할 수 있는 구조가 중요하다. 이 게임은 별, 로켓, 장애물이라는 직관적인 요소만으로 목표와 위험을 즉시 전달한다.

플레이어는 첫 화면에서 조작법을 확인하고 바로 게임에 진입한다. 별 하나를 먹으면 점수 1점이 오르고, 그 순간 로켓 속도와 장애물 압박이 조금씩 올라간다. 점수 획득이 보상이면서 동시에 다음 위험을 키우는 구조라서 “한 번 더” 플레이하게 만드는 짧은 세션형 재미에 집중했다.

## 3. 핵심 콘셉트

### 한 줄 설명

검은 우주 보드 안에서 빨라지는 로켓으로 별을 먹고, 우주 쓰레기와 소행성, 별똥별을 피하는 고속 생존 게임.

### 플레이 감각

- Snake처럼 보드 밖으로 나갈 수 없는 제한 공간
- Dodge 게임처럼 날아오는 장애물을 피하는 반사 신경
- 점수 상승에 따라 속도가 붙는 러너 게임의 긴장감
- 흑백 도형만으로 구성한 레트로 아케이드 분위기

## 4. 주요 규칙

- 별은 항상 1개만 존재한다.
- 별을 먹으면 점수 +1이 적용된다.
- 새 별은 기존 별을 먹은 뒤 다른 위치에 생성된다.
- 로켓은 정사각형 보드 밖으로 나갈 수 없다.
- 벽, 우주 쓰레기, 소행성에 충돌하면 목숨이 1 감소한다.
- 일반 별똥별에 닿으면 목숨이 0.5 감소한다.
- 반짝이는 회복 별똥별을 먹으면 목숨이 1 회복된다.
- 목숨은 최대 3이며, 0이 되면 Game Over다.
- 9999점에 도달하면 END 화면으로 장기 클리어를 표시한다.

## 5. 난이도 설계

난이도는 점수에 따라 자연스럽게 상승한다. 단순히 장애물 수만 늘리는 방식이 아니라, 로켓 속도와 장애물 속도, 등장 빈도를 함께 올려 사용자가 점점 빨라지는 흐름을 체감하도록 구성했다.

### 우주 쓰레기

- 게임 초반부터 등장하는 기본 장애물
- 보드 바깥에서 생성되어 직선으로 반대 방향까지 이동
- 점수 70점 이후부터 한 번에 최대 2개까지 등장
- 점수 150점 이후부터 한 번에 최대 3개까지 등장
- 점수가 높아질수록 이동 속도와 등장 빈도가 증가

### 소행성

- 100점 이상부터 등장
- 우주 쓰레기보다 크고 더 빠르다
- 플레이어를 지나치게 직접 겨냥하지 않도록 생성 방향을 조절한다
- 고득점 구간에서 회피 동선을 크게 흔드는 역할을 한다

### 별똥별

- 보드 모서리에서 반대쪽 모서리까지 대각선으로 가로지른다.
- 일반 별똥별은 가장 빠른 장애물보다 더 빠르게 이동한다.
- 별똥별은 아주 가끔 등장해 긴장감을 만든다.
- 회복 별똥별은 더 드물게 등장하며, 위험을 감수할 보상 선택지를 제공한다.

## 6. 시각 디자인

디자인은 검은 배경과 흰색 계열 도형만 사용한다. 별, 로켓, 장애물, HUD를 모두 흑백 톤으로 제한해 한눈에 읽히는 미니멀 아케이드 스타일을 만들었다.

- 검은색 정사각형 보드
- 흰색 로켓과 불꽃 파티클
- 작은 배경 별과 속도선
- 대각선으로 흐르는 별똥별
- 직선형 우주 쓰레기와 불규칙한 소행성 실루엣
- SCORE와 LIFE 중심의 간결한 HUD

## 7. 화면 구성

- Start: 게임명, 조작법, 시작 안내
- Play: 정사각형 보드, 로켓, 별, 장애물, SCORE, LIFE
- Pause: 일시정지 상태와 재개 안내
- Game Over: 최종 점수와 재시작 안내
- END: 9999점 도달 시 클리어 메시지

## 8. 기술 구조

프로젝트는 Canvas 기반으로 구성해 외부 이미지 에셋 없이 도형과 파티클만으로 게임을 렌더링한다. 게임 루프, 입력, 충돌, 난이도, 오브젝트, 이펙트를 역할별 폴더로 분리해 구조를 단순하게 유지했다.

```text
src/
  main.ts
  game/
    core/
      Game.ts
      Input.ts
      Collision.ts
      Difficulty.ts
      types.ts
    objects/
      Player.ts
      Star.ts
      Obstacle.ts
      Asteroid.ts
    effects/
      Background.ts
      Particle.ts
  ui/
    HUD.ts
    screens.ts
  styles/
    global.css
```

- 짧은 플레이, 빠른 재시작, 점수 경쟁에 적합한 구조
