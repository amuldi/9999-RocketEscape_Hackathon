Web Mini Game Challenge Proposal

9999:
Rocket Escape

검은 우주 보드 안에서 빨라지는 로켓으로 별을 먹고, 장애물을 피하며 오래 살아남는 흑백 고속 생존 미니게임.

실제 게임 플레이 화면: 로켓, 별, 우주 쓰레기, HUD가 모두 보드 안에서만 표시된다.

GitHub: https://github.com/amuldi/9999-RocketEscape_Hackathon
Deploy: https://space-rocket-game-smoky.vercel.app

Chapter 1. 게임 소개

9999: Rocket Escape는 정사각형 우주 보드 안에서 로켓을 조종해 별을 먹고 장애물을 피하는 Canvas 미니게임이다. Snake처럼 보드 밖으로 나갈 수 없고, 점수가 오를수록 로켓과 장애물이 함께 빨라진다.

장르흑백 레트로 아케이드 생존 게임
플랫폼웹 브라우저 / HTML Canvas
조작방향키 또는 WASD
목표START 후 오래 생존하고 로컬 랭킹에 도전
한 줄 콘셉트
검은 우주 보드 안에서 빨라지는 로켓으로 별을 먹고, 우주 쓰레기와 소행성, 별똥별을 피해 최고 점수에 도전한다. 닉네임 입력 후 START로 게임을 시작하고, 종료 후 점수가 로컬 랭킹에 저장된다.

Chapter 2. 핵심 재미

Speed
점수가 곧 속도다 별 획득, 생존 시간, 콤보가 점수로 이어진다. 보상과 위험이 동시에 커져 짧은 플레이에서도 긴장감이 생긴다.
Survival
좁은 보드 안의 회피 화면 밖으로 나갈 수 없기 때문에 장애물을 피하는 동선 선택이 중요하다.
Readability
흑백만으로 읽히는 규칙 별은 점수, 장애물은 위험, 반짝이는 별똥별은 회복이라는 의미를 움직임과 형태로 구분한다.
Retry
랭킹으로 다시 도전 로그인 없이 닉네임 기반 로컬 랭킹을 저장해 같은 브라우저에서 기록을 바로 비교한다.

Chapter 3. 플레이 규칙

요소 규칙
별 항상 1개만 생성된다. 먹으면 기본 점수와 콤보 보너스를 받고 새 위치에 다시 생성된다.
점수 별 획득 점수, 생존 시간 점수, 콤보 보너스를 합산한다.
로켓 방향키 또는 WASD로 움직인다. 보드 밖으로 나갈 수 없다.
목숨 최대 3개다. 벽, 우주 쓰레기, 소행성 충돌 시 1 감소한다.
별똥별 일반 별똥별에 맞으면 목숨 -0.5, 반짝이는 별똥별을 먹으면 목숨 +1이다.
종료 목숨이 0이면 Game Over, 9999점에 도달하면 END 화면을 보여주고 로컬 랭킹에 등록한다.

Chapter 4. 장애물과 난이도

우주 쓰레기

초반부터 등장하는 기본 장애물이다. 보드 바깥에서 생성되어 직선으로 이동하며, 점수가 높아질수록 더 빠르고 자주 등장한다. 70점 이후에는 최대 2개, 150점 이후에는 최대 3개까지 한 번에 등장한다.

소행성

100점 이상부터 등장한다. 우주 쓰레기보다 크고 빠르며, 고득점 구간에서 가장 큰 위협이 된다. 다만 플레이어를 직접 겨냥하는 불합리한 생성은 피하도록 방향을 조절한다.

별똥별

보드 모서리에서 반대쪽 모서리까지 대각선으로 지나간다. 일반 별똥별은 가장 빠른 장애물보다 더 빠르게 움직이고, 회복 별똥별은 더 드물게 등장해 위험을 감수할 보상을 만든다.

Chapter 5. 화면과 연출

전체 화면은 흑백만 사용한다. 검은 보드는 우주 공간을, 흰색 도형은 로켓, 별, 장애물, UI 정보를 표현한다. 외부 이미지 없이 Canvas 도형과 파티클만 사용해 레트로한 인상을 유지한다.

  • 로켓 뒤로 이어지는 흰색 불꽃 파티클
  • 보드 안에만 보이는 배경 별
  • 아주 가끔 한 개씩 대각선으로 지나가는 별똥별
  • SCORE와 LIFE 중심의 간결한 HUD
  • Start, Pause, Game Over, END 상태 화면

Chapter 6. 기술 구조

프로젝트는 게임 루프, 오브젝트, 이펙트, UI를 역할별로 나누어 관리한다. 불필요한 파일을 제거하고 이 게임 실행에 필요한 구조만 남겼다. 

src/
  main.ts
  game/
    core/       Game, Input, Collision, Difficulty, types
    objects/    Player, Star, Obstacle, Asteroid
    effects/    Background, Particle
  ui/           Menu, HUD, screens
  styles/       global.css

Chapter 7. 보완할 점

현재 버전은 START 중심의 단순한 진입 흐름과 로컬 랭킹으로 구성되어 있다. 다음 단계에서는 플레이 지속성과 완성도를 높이는 기능을 보완할 수 있다.

  • 글로벌 랭킹 저장 기능으로 다른 사용자와 경쟁
  • 모바일 터치 조작을 추가해 접근성 확대
  • 효과음과 음소거 옵션으로 별 획득, 충돌, 회복 피드백 강화
  • 난이도 프리셋으로 초보자와 숙련자 모두 대응
  • 회복 별똥별의 반짝임을 더 명확히 해 순간 판별성 개선

Chapter 8. 추가 기능 아이디어

  • 랭킹 보드: 최고 점수와 플레이어 이름 저장
  • 결과 공유 화면: 최종 점수를 이미지나 텍스트로 공유
  • 스킨 시스템: 흑백 규칙을 유지한 로켓 외형 변경
  • 미션 모드: 특정 점수 도달, 무피격 생존 같은 짧은 목표 제공
  • 점수 구간별 배경 변화: 고득점 구간에서 별 밀도와 파티클 연출 강화
